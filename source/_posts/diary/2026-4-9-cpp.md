- 右值引用T&&是类型.
- 编译器将已命名的右值引用视为左值，而将未命名的右值引用视为右值。
- int && r = 10; r的类型是int&&，但如果表达式使用r，r类型是int，lvalue。
C++ 表达式的类型不会是引用类型，引用类型决定value categories
- int&和int&&在ir中都是ptr。
- 如果是模板T&& ！= 右值引用，用来保存value categories信息。

8. 完美转发。std::forward

- 恢复 T 推导时的“值类别”. T&，T&&
- 引用折叠
- remove_reference_t
```cpp
#include <type_traits>

template<typename T>
constexpr T&& my_forward(std::remove_reference_t<T>& x) noexcept {
    return static_cast<T&&>(x);
}

template<typename T>
constexpr T&& my_forward(std::remove_reference_t<T>&& x) noexcept {
    static_assert(!std::is_lvalue_reference<T>::value,
                  "bad forward");
    return static_cast<T&&>(x);
}
```
